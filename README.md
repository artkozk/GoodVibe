# Telegram Wish + Chat Bot

Русскоязычный Telegram-бот для:
- пожеланий `спокойной ночи` и `доброе утро`
- кнопочного управления (без ручных команд)
- общения в группе с учетом контекста и истории по людям
- реакции на voice/video и пожелания
- обучения через фидбек и импорт переписок

## Зачем нужен бот

Бот закрывает 3 задачи:
- стабильная рассылка теплых сообщений утром и ночью
- живое общение в группе без повторяющегося "ботского" текста
- накопление памяти по участникам: кто дружелюбен, кто грубит, как отвечать каждому

## Что умеет бот

### Пожелания
- `🌙 Спокойной ночи` и `☀️ Доброе утро`
- режимы: `коротко`, `стандарт`, `по смыслу`
- отправка: в личку, в группу, в личку + группу
- авторассылка:
- 23:10 — ночь
- 07:00 — утро

### GPT + fallback
- генерация через OpenAI-совместимый API (в том числе io-cloud)
- при таймаутах/ошибках автоматом откат к локальным паттернам
- в статистике виден источник: `нейросеть` или `локальные паттерны`

### Групповое общение
- `💬 Общение в группе` вкл/выкл
- `🎚 Активность`:
- тихо
- норм
- каждое сообщение
- только вопрос
- ответы на "доброе утро"/"спокойной ночи"
- `🔥` реакция на voice/video
- `❤️` реакция на пожелания (чередуются обычные и premium при наличии ID)

### Социальная память по людям
- хранит рейтинг пользователя в группе (`-100 ... +100`)
- считает дружелюбные и грубые сигналы
- сохраняет причины обид и последний контекст
- режимы:
- `самообучение по людям` — тон ответа зависит от рейтинга/поведения
- `как я (по экспорту)` — сильнее копируется твой стиль
- админ может вручную поднять/опустить рейтинг через кнопки
- рейтинг админа защищен: у основного админа отношения не уходят в минус

### Подколы/обзывалки
- отдельный список дружеских слов
- просмотр, добавление, сброс через меню
- используется аккуратно в групповом общении

### Полный лог переписок
- бот пишет в лог все входящие сообщения, которые видит
- хранит `chat_id`, `message_id`, `user_id`, `username`, имя, тип контента, текст, время
- лог используется для анализа контекста и экспорта
- в админке есть экспорт переписки по каждому пользователю

### Поиск через `@v`
- в группе или в личке напиши: `@v <запрос>`
- пример: `@v сколько сейчас времени в амстердаме`
- бот вернет краткий ответ

### Импорт стиля общения
- `📥 Импорт стиля общения`
- поддержка:
- `JSON` экспорт Telegram
- `TXT` (по фразе на строку)
- `HTML` экспорт Telegram (`messages.html`)
- примеры используются в режиме группового общения

## Полностью кнопочный сценарий

Основная работа:
- `/start` в личке
- `⚙️ Настройки`
- включить нужные режимы
- генерировать через кнопки меню

Фидбек по качеству:
- `✅ Понравилось всё`
- `✨ Отметить хороший оборот`
- `🙁 Не понравилась часть`

Дополнительно:
- `🗨 Личка для всех` — режим обычного общения с ботом для неадминов в личке
- `📤 Экспорт переписок` — выгрузка лога по выбранному пользователю

## Установка

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python bot.py
```

## Конфиг (`user_config.py`)

Обязательное:
- `BOT_TOKEN`

Можно оставить `0`:
- `ADMIN_USER_ID` (автоопределение при первом `/start`)
- `GROUP_CHAT_ID` (можно закрепить кнопкой из группы)

Параметры GPT:
- `OPENAI_ENABLED`
- `OPENAI_API_KEY`
- `OPENAI_BASE_URL`
- `OPENAI_MODEL`
- `OPENAI_TEMPERATURE`
- `OPENAI_MAX_TOKENS`
- `OPENAI_TIMEOUT_SEC`
- `OPENAI_RULES`

Прочее:
- `TIMEZONE` (по умолчанию `Europe/Moscow`)
- `PREMIUM_EMOJI_IDS`
- `TRAINING_TARGET_SAMPLES`
- `CHAT_LOG_PATH` — путь к файлу лога сообщений (`.jsonl`)
- пути служебных JSON-файлов

## Права и настройки Telegram

- добавь бота в группу
- выдай права на отправку сообщений и реакций
- в BotFather отключи privacy mode (`/setprivacy -> Disable`), если бот должен видеть все сообщения

## Как понять, что работает нейросеть

- в логах есть строка:
- `Wish generation source=openai ...`
- в статистике:
- `из нейросети`
- `из локальных паттернов`

## Структура проекта

- `bot.py` — основной runtime
- `bot_state.py` — состояние, режимы, рейтинг людей
- `openai_wishes.py` — запросы в OpenAI-совместимый API
- `wishes.py` — локальная генерация
- `feedback_store.py` — хранение фидбека/статистики
- `adaptive_model.py` — локальная адаптивная модель
- `user_config.py` — конфиг без `.env`
